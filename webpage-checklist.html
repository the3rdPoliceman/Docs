Webpage Checklist
( See https://codeburst.io/the-front-end-checklist-8b2292fdda44 for more detail)
HTML
doctype 
<!doctype html>
html[lang] set to correct language
<html lang="fr">
rtl
<html dir="rtl">
encoding set
It's important that this is as high as possible up the page. Once the browser reads this, it will parse the whole document again from the top 
<meta charset="utf-8">
alternate languages
<link rel="alternate" href="https://es.example.com/" hreflang="es">
IE Conditionals
Conditional comments are present for IE if needed.
Critical CSS Inlined
inline critical CSS: CSS which styles content that is immediately visible during pageload (“above the fold content”) is called “critical CSS”. It is embedded before your principal CSS call and between 
<style></style>
in a single line (minified).
Import order
All CSS files are loaded before any JavaScript files
title set 
<title>Page Title less than 65 characters</title>
description 
<meta name="description" content="Description of the page less than 150 characters">
author set
favicon
see https://css-tricks.com/favicon-quiz/ & realfavicon webpage and https://github.com/audreyr/favicon-cheat-sheet e.g. 
<link rel="icon" type="image/png" href="https://example.com/favicon.png">
x-ua-compatible
<meta http-equiv="x-ua-compatible" content="ie=edge">
viewport
<meta name="viewport" content="width=device-width, initial-scale=1">
SRI (Subresource Integrity)
